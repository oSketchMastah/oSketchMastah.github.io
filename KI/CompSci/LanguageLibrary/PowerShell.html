Windows PowerShell

Powershell is a high-level command shell serving as a good alternative to CMD. This page is designed to teach you the powershell language while helping you to better understand its capabilities.

Fundamentals

  • Output is extensible: Modules with additional cmdlets and providers can be created
  • Handles console input and display: No effort is exerted parsing arguments or formatting output.
  • Uses some C# syntax: PowerShell is built on the .NET framework and shares syntax features and keywords of C#

The Naming System

PowerShell takes advantage of abstraction to create a relatively simple naming system

  • Number of names reduced by abstracting conceptually similar operations on processes
  • Cmdlets use Verb-Noun formatted names
  • The hyphen '-' is the standardized parameter indicator.

Get-Command -{Verb, Noun} Specifier: can be used to find cmdlets in PowerShell, returns a list of all cmdlets using the specified noun/verb.

Variables and cmdlets

Variables are specified with form $identifier ; A $ symbol followed by an identifer made up of letters and underscores.
Values are assigned using the = operator followed by a value, having form:

  • $id = value
Instantiate variables using a dollar sign followed by an identifer and the assignment operator with set value.
  • syntax: $id = value
  • Possible types include strings, files, directories and arrays of these things specified using just commas between items.
Pass variables/values to cmdlets (that have the -inputObject switch) using the pipeline operator '|' (won't work otherwise)
  • $var | cmdlet

Files and IO

Following are some switches available for many commands:

  • -Path SourcePath : Switch that specifies command to be performed on item specified as path by SourcePath.
  • -Destination DestPath : Switch that specifies path destination of output file/directory as DestPath.
  • -Recurse : When command has a well-defined source folder (specified by -path), switches to recursively perform command on all contained files and all sub-directories.
  • -Filter WcS : Used with -Recurse switch to limit the files on which command is performed to those satisfying the form given by the Wildcard Specifier (WcS)

** -path and -destination only need be specified to be unambiguous or for explicit specification of role of following path, which is necessary when going against the standard formatting for arguments shown

following are some functions for working with files, directories and IO
Creating File/Directory
New-item -ItemType ItemType: Creates an item with the specified type (File, Directory). Aliased w/ "ni".

  • Used with -path
Copy-item : Use for copying Files/Directories. Aliases include: copy, etc.
  • Used with -Path, -Destination, -Recurse, -Filter.
Remove-item : Use for deleting Files/Directories. Aliases include: erase, rm, ri, etc.
  • Used with -Path, -Recurse
Move-item : Use for moving Files/Directories. Aliases Include: move, mi, mv.
  • Used with -Path (folder or directory), -Destination (directory).
Rename-item : Use for renaming Files/Directories. Aliases Include: ren, rni
  • Used with -Path (folder or directory), -NewName (directory).
Test-path : Use to check if a folder/file exists. No aliases
  • Used with -Path, -Filter
Get-content : Use to retrieve the content of a file. Aliases are gc, cat, type.
  • Used with -Path (file)
  • Accessible Members include: length
Set-content : Use to set the content of a file. Aliased with "sc".
  • Used with -Path (file), -Value (content), -Filter.
Clear-content : Use to clear the content of a file. Aliased with "clc".
  • Used with -Path (file), -Filter.
Add-content : Use to add content as new line in a file. Aliased with "ac".
  • Used with -Path (file), -Value (content), -Filter.
