CMD - Windows Command-line

the windows CMD is the built in shell for making administrator commands to the operating system. Using this operating system (through CMD), one potentially has full access to the capabilities of ones computer. However CMD provides a set of functions that each access the operating system in a well-defined way, so one must know the possibilities offered by CMD.

Vanity Settings/Commands

One can change the way their CMD shell looks in a variety of ways.

  • cls : clears the shell of all previous output
  • CMD Propeties : change colors etc. by right-clicking the CMD window and entering the property settings

Key Commands

One can interact with the command-line simply using Ctrl + (some key). Following is a list of commands invoked this way:

  • Ctrl + C : cancels a function that is being run contemporarily
  • tab : auto-completes file paths/cycle through directiories
  • Ctrl + M : Enables Mark mode (cursor can be moved using arrow keys)
  • Ctrl + C/V : copy and pasting (Windows 10+)
  • Ctrl + F : Searches for text in the command prompt
  • Ctrl + Shift + mouse/ (+/-) keys : increases/decreases window transparency
  • Alt + Enter/ F11 : Makes the command prompt full screen
  • F1 : Retypes the last entered command letter-for-letter
  • F2 : copies the current command up to a specified character
  • F3 : retypes the entire previous line entered
  • F4 : deletes up to a character specified by cursor position
  • F5 : retypes previous command - letting you cycle through history (complex F3)
  • F6 : inserts Ctrl+Z(^Z) in the command prompt, an end-of-file indication
  • F7 : opens a list of previously entered commands to select from
  • F8 : Like F5 but more complex
  • F9 : retype a previous command by entering its line number

Command Parameters and Wildcards

Parameters to commands in CMD are generally specified using one of the characters '-', '+', or '/' followed by an abbreviation hinting at implied specification.

Wildcards in CMD specify unknowns portions of argument. Used for defining sets VIA defining the variable part of a set of strings/files/etc:

  • an asterisk '*' replaces an unknown string of characters
  • a question mark '?' replaces a single unknown character

Operators and Fundamentals

Following are some inline operators:

  • Command & Command : runs the second command successive to the first
    • && : First command successful -> Second command runs
    • || : First command unsuccessful -> Second command runs
  • Command | Command : referred to as a "pipe", passes the output of the first command as input to the second command
  • Command > FilePath: Saves output of command in file specified by FilePath
  • Command < FilePath: Extracts contents of file specified by FilePath to be used as input for the command

There is some constructs that can be put to use in almost any context viable to CMD, following are some fundamental commands:

  • if [NOT] Cond: used in batch files, runs if the condition specified by Cond is true:
    • NOT: switches block to execute if condition is false
    • span class="type">Cond: specified using comparison operators (LST, LEQ, EQU, GEQ, GTT)
  • for ObjectSwitch %var_name in Set do ...: defines a standard for loop where where objects are aliased as %var_name, objects have type specified by ObjectSwitch and physical set by Set.

    The syntax varies by the ObjectSwitch used, information is sorted accordingly:

    • netstat Specifiers {>=0}: Returns protocol-specific networking statistics.
      • -a : displays TCP connections that are active or with the listening state, as well as UDP ports that are being listened to
      • -b : lists filenames of all processes with an established network connection
      • -o : lists process identifiers (PIDs) of all processes with an established network connection
      • -e : lists statistic about network connection such as: bytes, unicast and non-unicast packets, discards, errors, and unknown protocols received or sent since connection was established
      • -f : forcefully lists Fully Qualified Domain Name (FQDN) of all foreign IP addresses when possible
      • -n : prevents netstat from attempting to determine host names of foreign IP addresses (time reduction)
      • -s : lists network statistic by protocol (use before -p if hoping to limit protocols)
      • -p Protocol : lists net stats for only a particular protocol
        • Protocol can be specified as one of tcp, udp , tcpv6 , or udpv6

      • -r : displays IP routing table. (same as using route command to execute route print
      • -t : switch to display TCP chimney offload state in place of typical TCP state
      • -x : lists all NetworkDirect listeners, connections, and shared endpoints
      • -y : shows TCP connection template for all connections (cannot be used with any other specifiers)
      • SecondsTime : specifies a time interval in between which the netstat command is automatically executed until it is terminated using Ctrl-C
      • I? : A keyword argument reserved for displaying information about the calling command (netstat)
